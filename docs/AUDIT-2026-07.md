# Audit & Improvement Notes — July 2026

This document records the findings from a structured review of the repository (four parallel
research passes: subagent architecture, game-dev domain accuracy, repo/script hygiene, and
distribution/installability) and tracks what this PR changed versus what remains as a roadmap.

## Headline finding

The 12 agents were high-quality **prose personas** but shipped **without the YAML frontmatter
Claude Code requires**, so none of them could be installed or auto-delegated to as native
subagents. The only documented usage was the anti-pattern `claude "Read agents/X.md and ..."`,
which is roleplay, not delegation. Making them real subagents — and installable — was the
single highest-leverage change and is the backbone of this PR.

## What this PR changed

### Installability (new)
- Added YAML frontmatter (`name`, `description`, `tools`, `model`, `color`) to all 12 agents.
  Descriptions are action-oriented with explicit "Use when / Use PROACTIVELY" triggers to
  drive automatic delegation. Tools are scoped per role (analysts get `WebSearch`/`WebFetch`
  and no `Bash`; developers get `Bash`; QA is near read-only; no agent gets `Agent`, so none
  can spawn nested subagents).
- Added a Claude Code **plugin manifest** (`.claude-plugin/plugin.json`) and an in-repo
  **marketplace** (`.claude-plugin/marketplace.json`) so the repo installs with
  `/plugin marketplace add … && /plugin install gamestudio-subagents@gamestudio`.
- Added `install.sh` / `install.ps1` fallbacks that copy the agents into
  `~/.claude/agents/` (or `./.claude/agents/` with `PROJECT=1`).
- Added `INSTALL.md` documenting all install paths.

### Correctness fixes (scripts)
- **Unreal projects generated a Godot folder layout.** `create_project_structure` tested
  `engine == "Unreal"` while the intake flow stores `"Unreal Engine"`, so the branch never
  fired. Now accepts both. (`scripts/init_project.py`)
- **`test_engine_system.py` failed 2 of 4 suites** — it called the `get_*_structure` methods
  with no `project_name`, and looked for generated engine files at the wrong path / with a
  stale engine name. Fixed; the suite is now **4/4 green**, which also makes the
  `SYSTEM_VERIFICATION.md` "100% pass rate" claim true.
- `package.json` scripts now invoke `python3` (was `python`, which fails where only `python3`
  exists). Fixed a duplicated "5." step number in the init summary.

### Dead references
- `producer_agent_v2.md` was referenced 9 times but the file is `producer_agent.md`
  (`EXAMPLES.md`, `SETUP.md`). Repointed.
- README "Documentation" links pointed at non-existent files (`example_workflows.md`,
  `templates/README.md`, `docs/api.md`). Repointed to real files; added `templates/README.md`.

### Factual corrections / broken code
- **De-pinned engine versions.** Agents/configs hardcoded "Godot 4.4.1" (7 files),
  "Unity 2023.2 LTS" (never an LTS), and "Unreal 5.3". Prose now says "latest stable Godot 4.x";
  configs updated to current stable (Godot 4.6, Unity 6 LTS `6000.0`, Unreal 5.6) with a
  standing "verify current version" note.
- **Shader wouldn't compile.** `uniform vec4 color : hint_color` → `source_color` (renamed in
  Godot 4); the fenced block is now `gdshader`, not `glsl`. (`agents/technical_artist.md`)
- **Particle API error.** `particles.process_material.emission.set("amount", …)` → `amount`
  is a property of the `GPUParticles2D` node: `particles.amount = …`. (`agents/technical_artist.md`)
- **Screen shake was non-functional** (infinite `set_loops` tween chaining offsets over 1/60 s).
  Replaced with the standard decaying-**trauma** model (offset via `camera.offset`,
  `FastNoiseLite` for smooth motion, `shake = trauma²`). (`agents/game_feel_developer.md`)
- Added a **Core Game-Feel Techniques** section with concrete defaults: coyote time
  (80-120 ms), input buffering (100-150 ms), variable jump height, hitstop (2-5 frames),
  squash & stretch, SFX pitch randomization.

## Orchestration: important design note

The repo is built around a `master_orchestrator` that "activates" and coordinates the other
agents. That model does not map cleanly onto Claude Code subagents:

- A subagent runs in an **isolated context** and returns only a final summary — there is no
  live shared state or "communication matrix" between sibling agents (the filesystem is the
  one channel that persists).
- Subagents **cannot use `AskUserQuestion`**, so the orchestrator's interactive intake
  interview can't run from inside a subagent.
- Coordination belongs at the **top level**. The recommended pattern is: the main session (or
  a `/new-game` slash command / skill) runs the intake, writes `project-config.json`, then
  delegates to the specialists **in sequence**, passing each the relevant context.

This PR keeps `gamestudio-orchestrator` installable (it is still useful as a planning
playbook) but its frontmatter and `INSTALL.md` reframe it as a top-level coordinator rather
than a runtime that dispatches other agents.

## Roadmap — now completed in this PR

A follow-up commit on this branch addressed the deeper enrichment items:

- **Added a `/new-game` command** (`commands/new-game.md`) that runs the intake interview at
  the top level and chains the specialists in sequence — the orchestration fix in practice.
- **Trimmed the mega-prompts** by moving report scaffolds into `templates/`: producer
  604→260, market_analyst 570→128, data_scientist 585→248 lines (new
  `production_report_template.md`, `market_report_template.md`, `analytics_report_template.md`).
- **Genericized QA**: removed the asteroid/space-shooter sample cases; added acceptance-criteria
  test derivation, GUT / gdUnit4 + `godot --headless`, soak/endurance, frame-time percentiles,
  input-latency, and save/load corruption tests; separated functional QA from usability playtesting.
- **Market analyst**: added named data sources (SteamDB/SteamSpy, Sensor Tower/data.ai, Newzoo,
  GameDiscoverCo), the wishlist KPI, Steam cut/refund facts, and an anti-fabrication guardrail.
- **Data scientist**: added genre-caveated retention/LTV benchmarks, an LTV definition, and A/B
  discipline (up-front sample size, α=0.05, anti-peeking/p-hacking).
- **Godot-native depth** added to mechanics (EventBus/FSM/composition/`move_and_slide`), UI/UX
  (anchors + `content_scale_mode`, focus nav, touch targets), art (atlasing, 9-slice, pixel-art),
  and technical art (renderers, `WorldEnvironment`, VRAM compression).
- **`engine_setup_template.md`**: replaced the serialized `InputEventKey` blob with Input Map
  guidance; migrated the Unity analytics snippet to Unity Gaming Services.

### Still open (future work)

Nice-to-haves beyond this PR:

- **Publish to a public marketplace** (e.g. submit to the community plugin catalog) so users can
  install without adding the repo as a marketplace first.
- **Automated agent linting in CI** — a small check that every `agents/*.md` has valid
  frontmatter (`name` + `description`) so future edits can't silently break installability.
- **Per-engine code branches** in the craft agents — the depth added here is Godot-first; Unity
  and Unreal equivalents (C#/Blueprint idioms) would make the multi-engine promise fully even.
